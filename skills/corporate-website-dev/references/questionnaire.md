# 需求调研问卷

在编写任何方案或代码之前，**必须**使用 Codex 提问工具逐项提问并记录答案。若环境提供 `/question` 或 `question.ask` 等专用指令，每个问题都要通过该工具发送，绝不能直接聊天询问。问题顺序不可打乱，以保证上下文完整。

## 1. 业务背景（自由文本）
- **提问内容**：「贵司属于哪个行业？请描述本次官网要聚焦的核心业务、目标受众以及独特价值主张。」
- **目标**：确定行业/领域、产品或服务重点、目标人群与关键差异化，并识别是否存在合规或信息披露要求。

## 2. 需要的页面（多选）
使用提问工具的多选模式，提供以下选项并允许选择「自定义页面」填写名称：
- 首页 / Hero
- 关于我们
- 产品或解决方案概览
- 功能亮点 / 价值主张
- 定价
- 成功案例 / 客户见证
- 博客 / 洞察
- 资源 / 下载中心
- 招聘 / 加入我们
- 联系我们
- 常见问题（FAQ）
- 自定义页面（填写名称）

## 3. 官方色彩搭配（自由文本或多选）
- **提问内容**：「请提供官网需要遵循的官方色彩方案，包括背景色、主色（Primary）以及任意辅色/强调色。若有十六进制或品牌手册请直接给出。」
- **执行方式**：使用提问工具的自由文本模式；如用户只有有限选项，可改用多选但需写明颜色名称或数值。
- **目标**：获取背景基色、主色、辅助/强调色、对比色或渐变方案，为后续设计令牌使用。

## 4. UI 风格（多选）
继续使用多选模式，让用户可组合多种风格：
- 玻璃化（Glassmorphism）
- 黏土化（Claymorphism）
- 极简主义（Minimal）
- 粗犷主义（Brutalism）
- 拟物化（Skeuomorphism）
- Bento 网格
- 暗黑模式
- 其他（描述期望）

## 使用提示
- 任一多选题若包含「自定义/其他」，立刻用提问工具追加自由文本追问获取细节（颜色值、页面目标等）。
- 对陌生的行业术语或色彩名称要请用户进一步解释，避免误解后续文案或设计。
- 若用户自发补充字体或其他品牌规范，也要记在工作区，但无需主动提问字体。
- 将整理后的答案保存，用于编写项目文档、信息架构和设计规范时引用原话。
