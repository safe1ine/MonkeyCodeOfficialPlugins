### RTLCSS Output Example

Source: https://getbootstrap.com/docs/5.3/getting-started/rtl

This example illustrates the CSS output generated by RTLCSS for the custom RTL font weight. It shows the default `bootstrap.css` with the original font weight and `bootstrap.rtl.css` with the overridden RTL font weight.

```css
/* bootstrap.css */
dt {
  font-weight: 700 /* rtl:600 */;
}

/* bootstrap.rtl.css */
dt {
  font-weight: 600;
}

```

Source: https://getbootstrap.com/docs/5.3/getting-started/rtl

This is a complete HTML starter template demonstrating the required HTML structure for enabling RTL in Bootstrap. It includes the `dir="rtl"` and `lang="ar"` attributes on the `<html>` tag, and links to the RTL Bootstrap CSS and JavaScript bundles.

  <head>
    <!-- Required meta tags -->
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">

    <!-- Bootstrap CSS -->
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.rtl.min.css" integrity="sha384-CfCrinSRH2IR6a4e6fy2q6ioOX7O6Mtm1L9vRvFZ1trBncWmMePhzvafv7oIcWiW" crossorigin="anonymous">

### Simultaneous LTR and RTL Imports with RTLCSS (Sass)

Source: https://getbootstrap.com/docs/5.3/getting-started/rtl

This Sass code uses RTLCSS directives to manage both LTR and RTL imports within the same stylesheet. By wrapping `@import` statements with `/* rtl:begin:options */` and `/* rtl:end:options */`, and defining a `stringMap` to swap 'ltr' with 'rtl', it allows for dual-direction support. The example imports Bootstrap's core styles within a `.ltr` class.

```scss
/* rtl:begin:options: {
  "autoRename": true,
  "stringMap":[ {
    "name": "ltr-rtl",
    "priority": 100,
    "search": ["ltr"],
    "replace": ["rtl"],
    "options": {
      "scope": "*",
      "ignoreCase": false
    }
  } ]
} */
.ltr {
  @import "../node_modules/bootstrap/scss/bootstrap";
}
/*rtl:end:options*/

```
